## 🗂️ 文档结构

每个版本目录包含：

```
docs/versions/vX.X/
├── README.md           # 版本概览（必须）
├── feature-plan.md     # 详细功能规划（推荐）
├── design-specs.md     # 设计规格（可选）
├── tech-specs.md       # 技术规格（可选）
├── test-plan.md        # 测试计划（可选）
└── assets/             # 相关资源（可选）
    ├── mockups/
    ├── diagrams/
    └── screenshots/
```

**约定：**
- ✅ 每个版本必须有 README.md
- ✅ 复杂版本建议创建独立目录
- ✅ 简单版本可以只在 ROADMAP 中描述

## 📝 版本创建指南

### 创建新版本文档

```bash
# 1. 创建版本目录
mkdir -p docs/versions/v1.X

# 2. 复制模板
cp docs/versions/v1.3/README.md docs/versions/v1.X/README.md

# 3. 编辑版本信息
# 修改版本号、状态、目标、功能清单

# 4. 更新本文件的版本索引
```

### 版本状态说明

- ✅ 已发布：版本已完成并发布到生产环境
- 🔄 进行中：正在开发中
- 📋 计划中：已规划但未开始
- ⏸️ 已搁置：暂时搁置
- ❌ 已取消：不再计划实施
